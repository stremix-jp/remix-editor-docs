# ファイル操作

remix-editor は複数のファイル形式に対応しています。

## インポート

### Remix JSON

![ファイルメニュー](./images/10-file-menu.png)

remix-editor の標準形式です。

**ファイル形式**: `.remix.json`

**含まれるデータ**:
- すべてのコントロールパターン
- パターン設定
- ポイントデータ

**インポート手順**:
1. **ファイル > インポート** を選択
2. または、ファイルをドラッグ＆ドロップ
3. インポート確認ダイアログで設定を確認
4. 「インポート」をクリック

**オプション**:
- **既存パターンを保持**: 既存のパターンを維持してマージ
- **正規化データとしてインポート**: 値の解釈方法を変更

### Funscript

The Handy などで広く使われる形式です。

**ファイル形式**: `.funscript`

**変換処理**:
- 時間: ミリ秒 → 秒
- 値: 0-100 → 0-maxValue

**オプション**:
- **上下反転**: 値を反転してインポート

### CSV（コントロールパターン）

汎用 CSV ファイルからパターンをインポートできます。

**詳細オプション**:

| オプション | 説明 |
|------------|------|
| データ方向 | 縦方向（列）/ 横方向（行） |
| 時間インデックス | 時間が入っている列/行番号 |
| 値インデックス | 値が入っている列/行番号 |
| 符号インデックス | 符号が入っている列/行番号（オプション） |
| 符号フォーマット | +/-, 1/-1, 0/1 など |
| 時間単位 | 秒/ミリ秒/デシ秒/センチ秒/分 |
| 値拡張 | 次のポイントまで値を保持 |
| ソース最大値 | マッピング用の最大値 |

### セクション CSV

セクションデータをインポートします。

詳しくは [セクション](./05-sections.md) を参照してください。

### ドラッグ＆ドロップ

ファイルをアプリケーション上にドラッグ＆ドロップすると、ファイル形式に応じて適切なインポートダイアログが開きます。

| 拡張子 | 処理 |
|--------|------|
| .remix.json | Remix JSON インポート |
| .funscript | Funscript インポート |
| .csv | CSV インポートダイアログ |
| .mp3, .wav など | 音声ファイル読み込み |

## エクスポート

### Remix JSON

remix-editor の標準形式でエクスポートします。

**手順**:
1. **ファイル > Remixを保存...** を選択
2. ファイル保存ダイアログで場所を指定
3. 保存

**ファイル名形式**: `{音声ファイル名}.{ISO8601タイムスタンプ}.remix.json`

**例**: `sample_audio.2026-01-15T10-30-00.remix.json`

**含まれるデータ**:
- すべてのコントロールパターン
- パターン設定（preferredActuatorType 含む）
- 全ポイントデータ

### セクション CSV

セクションデータを CSV としてエクスポートします。

**手順**:
1. **ファイル > セクション CSV エクスポート** を選択
2. ファイル保存ダイアログで場所を指定
3. 保存

**ファイル名形式**: `{音声ファイル名}.{ISO8601タイムスタンプ}.sections.csv`

**形式**: RFC 4180 準拠（適切なエスケープ処理）

## 自動保存

### 作業データの自動保存

remix-editor は以下のデータを自動的にブラウザに保存します：

| データ | 保存先 |
|--------|--------|
| コントロールパターン | LocalStorage |
| 音声ファイル | IndexedDB |
| 設定 | LocalStorage |
| セクションデータ | LocalStorage |

### 復元

次回起動時、保存されたデータが自動的に復元されます。

## キャッシュ管理

### 作業データのクリア

すべての作業データを削除して初期状態に戻せます。

**手順**:
1. **設定 > 詳細設定** を開く
2. 「作業データをクリア」をクリック
3. 確認ダイアログで「クリア」を選択

**削除されるデータ**:
- LocalStorage のすべてのデータ
- IndexedDB のすべてのデータ

**注意**: この操作は取り消せません。必要なデータは事前にエクスポートしてください。

## v1 ファイルの互換性

### 旧形式のサポート

remix-editor v1 で作成したファイルも読み込めます。

| v1 形式 | v2 での扱い |
|---------|-------------|
| `{x, y}` 形式 | 自動的に `{time, value}` に変換 |
| 正規化値 | 実値に変換 |

### 変換時の注意

- v1 のファイルを v2 で保存すると、v2 形式になります
- v1 形式に戻すことはできません

## バックアップのすすめ

### 定期的なエクスポート

重要な作業は定期的にエクスポートして保存することをお勧めします。

- ブラウザのデータはクリアされる可能性がある
- 別の PC で作業を続ける場合にも便利

### エクスポートのタイミング

- 大きな編集を終えた後
- 作業を中断する前
- 共有する前

## トラブルシューティング

### インポートに失敗する

1. ファイル形式が正しいか確認
2. ファイルが破損していないか確認
3. 別のファイルで試す

### エクスポートしたファイルが開けない

1. ファイル拡張子が正しいか確認
2. テキストエディタで内容を確認
3. JSON 構文エラーがないか確認

### データが復元されない

1. ブラウザの設定で LocalStorage が有効か確認
2. プライベートブラウジングでないか確認
3. ブラウザのデータがクリアされていないか確認
